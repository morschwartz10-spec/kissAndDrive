# Kiss & Drive - מערכת לניהול תורנויות הורים 🚗

מערכת ווב רספונסיבית מבוססת ענן (Firebase) שנועדה להחליף את ניהול טבלאות האקסל הידניות ולאפשר הרשמה, שיבוץ וניהול מהיר של מתנדבים לעמדת ה"נשק וסע" בבית הספר. האפליקציה פותחה בגישת Mobile First על מנת לאפשר להורים נוחות מקסימלית מהטלפון הנייד.

## תכונות עיקריות 🌟

- **לוח משמרות אינטראקטיבי:** תצוגה חודשית חכמה עם מקרא צבעים לזיהוי מהיר של חוסרים, כולל יכולת ניווט ("גלילה בקליק") לפירוט המשמרת.
- **זרימת הרשמה חלקה (Frictionless UX):** כפתור "התנדב למשמרת זו" המאפשר להורה מוכר (קיים במערכת) להשתבץ בתוך שנייה אחת רק על ידי הזנת מספר הטלפון שלו, ללא צורך במילוי טופס.
- **טופס התנדבות מתקדם:** מאפשר להורים לבחור ימים קבועים בשבוע או תאריכים ספציפיים, לבחור חודשים מועדפים, ולסמן עד 4 שכבות גיל של ילדיהם.
- **ניהול עצמי (Self Service):** אזור אישי המבוסס על מספר טלפון בלבד (ללא סיסמאות מורכבות), בו ההורה יכול לראות את משמרותיו העתידיות ולבטלן בקליק.
- **בסיס נתונים בזמן אמת:** מסונכרן באופן מיידי בין כל המשתמשים באמצעות Google Firebase Firestore.
- **אוטומציה לניקוי נתונים:** מנגנון מובנה (Rolling 3-Months Retention) למחיקת נתונים היסטוריים שעברו 90 יום, לשמירה על יעילות ופרטיות המסד.

## פאנל ניהול (Admin Panel) ⚙️

למערכת קיים פאנל ניהול (המוזן באמצעות גישה לכתובת `/index.html#admin`), המאפשר למנהל המערכת:
1. לצפות בכלל נתוני המתנדבים, מספרי הטלפון ושכבות הילדים.
2. לשלוח הודעת **WhatsApp** בלחיצת כפתור ישירות להורה.
3. לבטל שיבוצים.
4. לצפות ב**דוחות וסטטיסטיקות:** אלופי ההתנדבות ברבעון האחרון והשכבות המובילות בבית הספר.
5. איפוס מאגר נתונים מוחלט לקראת תחילת שנה (מוגן בשכבת אימות כפולה).

## טכנולוגיות (Tech Stack) 💻

* **Frontend:** HTML5, CSS3 (Vanilla), JavaScript (Vanilla) - ללא פריימוורקים לטובת ביצועים מהירים ופשטות. עיצוב Glassmorphism.
* **Backend & DB:** Firebase v8 Compat (Firestore).

## הוראות התקנה והרצה 🚀

כיוון שהמערכת נבנתה כ-Client-Side Application בלבד, ההרצה היא פשוטה ואינה דורשת שרת Node.js מקומי:

1. **פתיחת בסיס נתונים:**
   - פתחו פרויקט חדש ב- [Firebase Console](https://console.firebase.google.com/).
   - הפעילו את שירות ה-Firestore Database (במצב Test Mode או עם חוקי קריאה/כתיבה מותאמים).
   - צרו אפליקציית Web חדשה וקבלו את מפתחות ה-`firebaseConfig`.

2. **הגדרת המפתחות בקוד:**
   - פתחו את הקובץ `app.js`.
   - החליפו את אובייקט ה-`firebaseConfig` הנמצא בראש הקובץ במפתחות שקיבלתם מ-Firebase.

3. **הרצה מקומית:**
   - פשוט פתחו את הקובץ `index.html` בדפדפן ה-Chrome או ה-Safari שלכם.

4. **העלאה לאוויר (Deployment):**
   - האפליקציה מוכנה להעלאה חינמית ופשוטה לאוויר דרך **GitHub Pages**, **Vercel** או **Netlify**.
   - יש להעלות את שלושת קבצי הליבה (`index.html`, `style.css`, `app.js`) למחסן הגיטהאב שלכם ולהפעיל את שירות ה-Pages.
